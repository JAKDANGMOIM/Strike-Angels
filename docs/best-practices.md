# Strike Angels 모범 사례 가이드

이 가이드는 [구현계획서.md](../구현계획서.md)에 정의된 프로젝트를 개발할 때 따라야 할 웹게임(Phaser 3) 기반 모범 사례와 성능 최적화 팁을 제공합니다.

## Phaser 3 개발 모범 사례

### Scene 라이프사이클

- `preload()`: 에셋 로딩, 텍스처, 스프라이트 시트 다운로드 처리
- `create()`: 게임 씬 시작 시 한 번만 실행되는 초기화 로직 및 오브젝트 인스턴스화
- `update(time, delta)`: 매 프레임 호출, 입력 처리 및 타이머 기반 로직 등 처리

### 최적화 기법

- 오브젝트 그룹(Group) 및 풀링 사용: 총알, 적, GEM 등 자주 생성/삭제되는 오브젝트들은 메모리를 위해 Phaser Group의 `maxSize`와 풀링 메커니즘을 사용
- 물리 계통 최적화: `arcade` 물리를 기본으로 하여 복잡한 콜리전 연산을 피하고, AABB 충돌 연산 위주로 구성
- 스프라이트 시트(Texture Atlas): 여러 이미지 로딩 비용을 아끼기 위해 텍스처 아틀라스나 스프라이트 시트를 적극 활용

## 선 그리기 및 영역 점령 시스템 (웹 기반)

### Graphics 및 Geometry 최적화

- Phaser의 `Graphics` 객체 또는 캐싱된 Shape를 사용하여 점의 개수를 적절히 조절
- 렌더링 부하를 줄이기 위해 곡선보다는 최소 단위의 직선 위주로 설계하고, 해상도는 플레이에 영향 없는 선에서 조정

### 영역 채우기 알고리즘

- 플러드 필(Flood Fill) 등은 CPU 연산 자원을 많이 소모할 수 있으므로 웹워커(Web Worker) 또는 비동기/프레임 오프로딩으로 분산 처리 고려
- 캔버스의 특정 픽셀 데이터를 읽어오는 로직은 무거울 수 있으므로 추상화된 그리드 기반 연산 위주로 최적화

## 입력 처리

### 포인터(마우스/터치) 및 키보드 통합

- `scene.input.keyboard` 객체 활용
- 크로스 플랫폼을 고려해 모바일 터치(포인터 드래그, 화면 조이스틱 등)와 PC 키보드 양쪽이 잘 호환되도록 구성
- 이동 벡터의 길이를 정규화(`normalize()`)하여 대각선 이동 속도가 비정상적으로 빨라지지 않게 처리

### 이동 및 회전 보간법

- 물리 엔진(`setVelocity`)을 사용하여 이동을 처리하고, 점진적인 회전을 위해 Lerp를 수동으로 적용 (`currentAngle += diff * factor`)
- `update` 주기마다 delta 타임에 동기화되는 프레임 독립적인 움직임(`delta / 1000` 곱하기 등) 처리 보장

## 충돌 처리

### 물리 그룹과 콜라이더

- Arcade Physics의 `overlap` 및 `collider` 메서드를 활용
- 너무 많은 객체들이 콜리전 계산에 들어가지 않도록 `disableBody` 및 상태 `setActive(false)` 처리 철저
- 불필요한 레이캐스트보다는 바운딩 박스/반경 서클을 이용해 단순한 검사를 권장

## 데이터 관리 및 구조화

### JSON 및 설정 상수 활용

- 설정 데이터는 별도의 `JSON` 또는 TypeScript 상수로 분리해서 관리 (`src/config/skills.json` 등)
- 데이터가 정적이므로 메모리를 크게 신경 쓰기보다는 초기 로드나 번들에 패킹하여 빠르게 공급

### 컴포넌트 간 결합도 축소

- 씬(Scene) 이벤트 에미터(Extends EventEmitter)를 통해 상태 변동(적 사망 등)을 관리하고 전역 싱글톤은 지양
