# Strike Angels 구현 단계 가이드

이 가이드는 [구현계획서.md](../구현계획서.md)에 정의된 게임 개발 단계를 체계적으로 따르기 위한 규칙을 제공합니다.

## 개발 원칙

1. 각 단계는 순차적으로 진행합니다.
2. 컴포넌트 간에는 이벤트나 Phaser 씬 컨텍스트 넘기기를 통해 유기적으로 결합됩니다.
3. 작업 완료 시 구현계획서의 체크리스트를 업데이트합니다.
4. 각 컴포넌트는 최소 기능으로 먼저 구현한 후 점차 기능들을 확장합니다.

## 단계별 개발 접근법

### 1단계: 기본 시스템 및 플레이어 구현 (완료)

- `Player.ts`, `Bullet.ts` 기본 렌더링 및 물리 이동 로직 구성 완료
- 방향타를 통한 회전 및 전방 전진 시스템 완료
- 자동 공격(일정 주기로 총알 객체 풀링 발사)과 애프터버너 로직 완료

### 2단계: 선 그리기 및 영역 점령 시스템

- Phaser `Graphics` 혹은 `Rope` 기법 등을 활용하여 이동 동선에 따른 궤적 그리기 구성
- 선이 끊기지 않게 이어진 후 폴리곤 형태로 변환, 영역 내부 면적 계산(Grid 방식 또는 지오메트리)
- 게임의 메인 메커니즘이므로 최적화를 우선적으로 고려해 구성해야 합니다.

### 3단계: 적 AI 및 생성 시스템

- 추상화된 기본 추적형 적 패턴 먼저 작성
- `Group.get()` 을 활용해 시간에 따른 생성기(`EnemySpawner.ts`) 구현
- 총알과 적의 Arcade Physics `overlap`을 처리하여 데미지 로직 구현.

### 4단계: 경험치 및 스킬 시스템

- 몬스터 죽었을 때 GEM 엔티티 생성, 플레이어와 overlap 시 흡수(Tween) 처리
- 플레이어 레벨업 및 게임 씬 일시 정지(`scene.pause()`) 후 스킬 선택 화면(`UIScene`) 부착
- 선 그리기 효율화, 총알 데미지, 이속 등등의 JSON 데이터 기반 적용

### 5단계: UI 및 추가 시스템

- 체력 게이지, 미니맵(가능할 경우), 점령도 텍스트 그리기
- 게임 승리/패배 상태 시 씬 전환 (`this.scene.start('GameOverScene')`) 구성

### 6단계: 최적화 및 폴리싱

- 프로파일링을 통해 병목 구간(주로 플러드 필 점령 연산 등)을 발견하고 최적화 진행
- Phaser의 WebGL Particle 시스템 등을 활용해 화려한 2D 이펙트 부가
- 사운드 매니저 부착
